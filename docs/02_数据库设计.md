# 数据库设计

## 核心表

| 表名 | 说明 |
| --- | --- |
| `sys_user` | 系统用户表 |
| `sys_role` | 角色表 |
| `sys_permission` | 菜单与按钮权限表 |
| `sys_user_role` | 用户角色关系表 |
| `sys_role_permission` | 角色权限关系表 |
| `community` | 小区基础信息表 |
| `building` | 楼栋表 |
| `room` | 房屋表 |
| `resident` | 住户表 |
| `repair_order` | 报修工单表 |
| `complaint` | 投诉建议表 |
| `fee_item` | 费用项目表 |
| `bill` | 账单表 |
| `payment_record` | 缴费记录表 |
| `parking_vehicle` | 停车车辆表 |
| `notice` | 公告活动表 |
| `community_activity` | 社区活动表 |
| `express_package` | 快递代收表 |
| `access_control` | 门禁设备表 |
| `patrol_task` | 巡检任务表 |

## 关系说明

- 用户通过 `sys_user_role` 绑定一个或多个角色。
- 角色通过 `sys_role_permission` 绑定菜单权限和按钮权限。
- 小区、楼栋、房屋、住户构成基础数据链路。
- 报修和投诉建议记录住户服务请求及处理结果。
- 费用项目用于定义收费标准，账单记录应收信息，缴费记录记录实收流水。
- 公告、社区活动和快递代收共同支撑智能服务模块。
- 门禁设备、停车车辆和巡检任务共同支撑安防管理模块。
- 统计模块从房屋、住户、报修、账单、投诉等业务表聚合生成运营指标。

## 进阶模块表设计

| 表名 | 关键字段 | 业务说明 |
| --- | --- | --- |
| `community_activity` | `title`、`activity_type`、`location`、`organizer`、`start_time`、`signups`、`status` | 记录社区活动、报名人数和活动状态 |
| `express_package` | `tracking_no`、`recipient_name`、`phone`、`pickup_code`、`cabinet_no`、`status`、`arrived_at`、`picked_at` | 记录快递到件、取件码、存放位置和领取时间 |
| `access_control` | `device_name`、`gate_name`、`device_type`、`status`、`manager`、`last_check_at` | 记录门禁设备位置、类型、负责人和在线状态 |
| `patrol_task` | `route_name`、`area`、`assignee`、`plan_time`、`status`、`result`、`finished_at` | 记录巡检路线、计划时间、执行人和完成结果 |

## 索引与规范

- 主键均使用 `BIGINT AUTO_INCREMENT`。
- 金额字段使用 `DECIMAL(10,2)`。
- 时间字段使用 `DATETIME`。
- 业务查询高频字段包括：账号、角色编码、权限编码、报修编号、投诉编号、账期、房号。
- 状态字段统一使用短文本枚举，例如：`报名中`、`已结束`、`待领取`、`已领取`、`在线`、`离线`、`待巡检`、`已完成`。

完整建表脚本见：`database/schema.sql`。
初始化数据脚本见：`database/init_data.sql`。
